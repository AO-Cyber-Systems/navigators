<anti_patterns>
- DO NOT use streaming RPCs for sync -- use batched unary RPCs with cursor pagination
- DO NOT process operations out of creation order -- always ORDER BY createdAt ASC
- DO NOT retry operations infinitely -- cap at 5 retries, then flag as dead-letter
- DO NOT sync during active field work with blocking UI -- sync is non-blocking background
- DO NOT send individual operations -- always batch (up to 50 per push)
- DO NOT trust client timestamps for conflict resolution -- server assigns authoritative timestamps
- DO NOT use sqlcipher_flutter_libs -- DEPRECATED, already handled by TRD 04-01
</anti_patterns>

<error_recovery>
- If push batch partially succeeds: server returns which operation IDs processed. Client dequeues only those, leaves rest as pending for retry.
- If network drops mid-push: operations marked in_progress are reset to pending on catch. Next sync cycle retries.
- If WorkManager isolate can't access DB: shareAcrossIsolates was set in TRD 04-01. If still failing, check IsolateNameServer registration.
- If connectivity_plus reports connected but internet is down: use internet_connection_checker_plus before sync attempt.
- If server returns conflict error for a specific operation: log the conflict, mark operation as failed with reason, increment retryCount. Do NOT retry conflicts automatically.
</error_recovery>

<research_context>
From 04-RESEARCH.md:
- Operation log / transactional outbox: every write records an entry in sync_operations table in the SAME database transaction as the actual data write. Sync engine reads this table to push.
- Push sync: read pending ops (ORDER BY createdAt ASC, LIMIT 50), mark in_progress, send PushSyncBatchRequest, dequeue processed IDs, reset failed to pending.
- Each sync operation has client-generated UUID (entity_id). Server uses INSERT ON CONFLICT keyed on this UUID for idempotency.
- Contact logs are APPEND-ONLY (no conflict). Survey responses also append-only. Voter metadata edits (tags, notes) use LWW with server timestamp.
- WorkManager: registerPeriodicTask with 15min frequency, NetworkType.connected constraint, requiresBatteryNotLow. callbackDispatcher is top-level @pragma('vm:entry-point').
- connectivity_plus: stream-based, listen for reconnect, trigger immediate foreground sync.
- Jitter on reconnect: random delay 0-60s to prevent sync storms when cell tower comes back in range.
- iOS caveat: background execution unreliable. Aggressive foreground sync is primary mechanism (sync on app open, on connectivity change, between screen navigations).
</research_context>

       func (s *SyncService) PushSyncBatch(ctx context.Context, userID, companyID uuid.UUID, ops []SyncOperation) (*PushSyncBatchResult, error) {
         var processed []string
         var errors []SyncError

         for _, op := range ops {
           // 1. Check idempotency -- skip if already processed
           exists, err := s.queries.CheckSyncOperationProcessed(ctx, op.ClientOperationID, companyID)
           if exists { processed = append(processed, op.ClientOperationID); continue }

           // 2. Process by entity type
           switch op.EntityType {
           case "contact_log":
             // Append-only: INSERT ON CONFLICT DO NOTHING
             err = s.processContactLog(ctx, companyID, userID, op)
           case "voter_metadata":
             // LWW: UPDATE WHERE updated_at < client_timestamp
             err = s.processVoterMetadata(ctx, companyID, op)
           default:
             errors = append(errors, SyncError{OperationID: op.ClientOperationID, Code: "unknown_entity", Message: "unknown entity type"})
             continue
           }

           if err != nil {
             errors = append(errors, SyncError{OperationID: op.ClientOperationID, Code: "processing_error", Message: err.Error()})
             continue
           }

           // 3. Record as processed for idempotency
           s.queries.RecordSyncOperationProcessed(ctx, ...)
           processed = append(processed, op.ClientOperationID)
         }
         return &PushSyncBatchResult{ProcessedIDs: processed, Errors: errors}, nil
       }
