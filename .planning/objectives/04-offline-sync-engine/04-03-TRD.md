<anti_patterns>
- DO NOT show blocking spinners during sync -- Navigator may be at a door. Use background sync and update UI reactively via Drift streams.
- DO NOT call server APIs for data that should be locally cached when offline. Walk list and voter list MUST read from local DB.
- DO NOT delete local data before pushing pending operations on turf reassignment -- push first, clean second.
- DO NOT force a full re-download on every app open. Check sync manifest for assignment changes, then do delta sync.
- DO NOT poll for sync status -- use Drift's reactive .watch() streams for zero-overhead UI updates.
</anti_patterns>

<error_recovery>
- If forced sync on app open fails (no internet): continue with stale local data, show "last synced X ago" in status.
- If turf download fails mid-batch: cursor is preserved from last successful batch. Next attempt resumes from cursor.
- If connectivity_plus reports connected but sync fails: show "Sync failed" in status, retry on next connectivity change.
- If database provider is not yet initialized when app starts: show a loading screen until DB is ready (key retrieval from flutter_secure_storage).
</error_recovery>

<research_context>
From 04-RESEARCH.md:
- Sync status: Riverpod StreamProvider watches sync_operations count via Drift .watch(). Exposes pendingOperations, lastSyncAt, isSyncing, lastError.
- Forced sync on app open: AppLifecycleListener triggers sync on resume; initial sync check in startup flow.
- Repository pattern: service layer reads from local DB first, returns cached data. Network calls update local DB, Drift's reactive streams auto-update UI.
- Turf reassignment: on sync, compare server assignments to local. For removed turfs: push pending ops first, then delete local data for that turf.
- Stale cursor detection: server returns cursor_valid: false when cursor is too old. Client resets and does full pull.
</research_context>

       Future<void> handleTurfReassignment(SyncManifest manifest) async {
         final localAssignments = await db.select(db.turfAssignments).get();
         final serverTurfIds = manifest.turfs.map((t) => t.turfId).toSet();
         final localTurfIds = localAssignments.map((a) => a.turfId).toSet();

         // Removed turfs: push pending, then clean
         final removed = localTurfIds.difference(serverTurfIds);
         for (final turfId in removed) {
           await pushSync.pushOperationsForTurf(turfId); // Push first!
           await VoterDao(db).deleteVotersForTurf(turfId);
           await (db.delete(db.turfAssignments)
             ..where((t) => t.turfId.equals(turfId))).go();
         }

         final added = serverTurfIds.difference(localTurfIds);
         for (final turf in manifest.turfs.where((t) => added.contains(t.turfId))) {

               turfName: turf.turfName,
               assignedAt: DateTime.now(),
               boundaryGeojson: turf.boundaryGeojson,
