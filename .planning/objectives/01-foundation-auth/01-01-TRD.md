func RunMigrations(databaseURL string, migrationsFS fs.FS) error {
    source, _ := iofs.New(migrationsFS, ".")
    m, _ := migrate.NewWithSourceInstance("iofs", source, databaseURL)
    if err := m.Up(); err != nil && err != migrate.ErrNoChange { return err }
    return nil
}

<anti_patterns>
- DO NOT fork eden-platform-go. Always consume via go.mod replace directive.
- DO NOT use pgvector/pgvector:pg16 for Navigators -- need postgis/postgis:16-3.4 image for spatial queries.
- DO NOT share schema_migrations table between eden and navigators migrations. Use separate tracking tables.
- DO NOT use B2C mode or allow public signup. Pre-create MaineGOP company, admin creates users.
- DO NOT put JWTKeySeedPath as empty string in production. For dev, empty = ephemeral key (fine).

<research_context>
From 01-RESEARCH.md:
- Eden-platform-go provides full auth (signup, login, JWT refresh, password hash) and RBAC (enforcer, interceptors, permission matrix).
- Consumer apps bootstrap by: (1) running eden migrations, (2) creating pgstore.Backend, (3) wiring auth/rbac services, (4) registering handlers on mux with interceptor chain.
- Dual migration strategy: eden uses default `schema_migrations` table; navigators must use a custom table name via golang-migrate's `x-migrations-table` query param in the database URL.
- Eden's config.Load() provides DATABASE_URL, SERVER_ADDR, NATS_URL, MINIO_ENDPOINT, JWT_KEY_SEED_PATH.
- PostGIS requires postgis/postgis:16-3.4 Docker image; first navigators migration should CREATE EXTENSION postgis.
</research_context>

<gotchas>
- golang-migrate pgx5 driver accepts both "postgres://" and "pgx5://" URLs but pgxpool only accepts "postgres://". Use "postgres://" everywhere.
- golang-migrate custom migration table: append `?x-migrations-table=schema_migrations_navigators` to the database URL passed to RunMigrations for navigators migrations. Eden's RunMigrations does NOT support this param -- you must create a navigators-specific RunMigrations wrapper or call migrate.New directly with the table name option.
- eden's `pgstore.NewBackend()` calls RunMigrations internally. For navigators migrations you must call RunMigrations separately AFTER pgBackend is created.
- The nats:2.11-alpine image does NOT enable JetStream by default. You must pass `--jetstream` in the command.
- eden-platform-flutter requires SDK ^3.11.1. Ensure Flutter version matches.
</gotchas>
