- Door knock UX flow: Walk List -> Tap voter -> Door Knock Screen -> Step 1: Disposition (4 big buttons: Not Home, Answered, Refused, Moved) -> If Not Home/Refused/Moved: quick save + auto-advance -> If Answered: Step 2: Sentiment (EdenRating 1-5) -> Step 3: Survey form (if active) -> Step 4: Notes (optional) -> Save All -> Return to walk list, advance to next
- Survey field types to support: single_select, multi_select, text, number, boolean -- map to EdenSelect, EdenMultiSelect, EdenInput, EdenInput(number), EdenToggle
- Survey forms are pull-synced from server. Check local Drift for active forms. If none, skip survey step.
- Note visibility default: 'team'. Allow override to 'private' or 'org' via EdenSelect.
</research_context>

- Survey forms must be pulled to local DB BEFORE they can be rendered. Ensure pull_sync fetches survey forms. If a form isn't cached locally, the survey step should gracefully skip or show a message.
