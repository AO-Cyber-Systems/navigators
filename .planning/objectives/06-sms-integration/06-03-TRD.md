<gotchas>
- ConnectRPC JSON protocol: POST with JSON body, not query params. Content-Type: application/json.
- Auth token from eden platform authProvider -- pattern established in existing services
- Timestamps come as ISO 8601 strings from proto JSON -- parse with DateTime.parse()
- Campaign segment_filters is a JSON string in proto -- pass through as-is from UI filter builder
- Template merge fields displayed as chips/tags in UI for clarity
- Admin-only screens (templates, campaigns) should check role before showing in nav
</gotchas>

1. Create `services/sms_service.dart`:
   - Models: SMSMessage (id, voterId, userId, direction, messageType, body, status, createdAt, fromNumber, toNumber), ConversationSummary (voterId, voterName, lastMessageBody, lastMessageAt), SMSTemplate (id, name, body, mergeFields, isActive, createdAt), SMSCampaign (id, name, templateId, status, totalRecipients, sentCount, deliveredCount, failedCount, launchedAt, completedAt, createdAt)
   - All models with `fromJson` factory constructors
   - Riverpod provider: `final smsServiceProvider = Provider((ref) => SMSService(ref));`
   - Methods (all async, all use ConnectRPC JSON POST pattern from voter_service.dart):
     - `sendP2PMessage(voterId, body)` -> SMSMessage
     - `getConversation(voterId, {pageSize, pageToken})` -> List<SMSMessage>
     - `listConversations({pageSize, pageToken})` -> List<ConversationSummary>
     - `createTemplate(name, body, mergeFields)` -> SMSTemplate
     - `listTemplates()` -> List<SMSTemplate>
     - `getTemplate(templateId)` -> SMSTemplate
     - `updateTemplate(templateId, name, body, mergeFields)` -> void
     - `deleteTemplate(templateId)` -> void
     - `previewTemplate(templateId)` -> String (rendered body)
     - `createCampaign(name, templateId, segmentFilters)` -> SMSCampaign
     - `launchCampaign(campaignId)` -> void
     - `pauseCampaign(campaignId)` -> void
     - `cancelCampaign(campaignId)` -> void
     - `getCampaign(campaignId)` -> SMSCampaign
     - `listCampaigns({pageSize, pageToken})` -> List<SMSCampaign>
     - `getSMSConfig()` -> SMSConfig
     - `updateSMSConfig(...)` -> void
     - `get10DLCStatus()` -> String

2. Create `features/sms/conversation_list_screen.dart`:
   - ConsumerStatefulWidget
   - Loads conversations via smsService.listConversations()
   - ListView of conversation tiles: voter name, last message preview (truncated to 1 line), timestamp (relative: "2m ago", "Yesterday")
   - Tap tile -> navigate to ConversationThreadScreen(voterId)
   - FAB: compose new message -> navigate to ComposeMessageScreen
   - Empty state: "No conversations yet" with icon
   - Pull-to-refresh
   - Pagination with scroll listener

3. Create `features/sms/conversation_thread_screen.dart`:
   - Takes voterId parameter
   - AppBar shows voter name
   - Loads messages via smsService.getConversation(voterId)
   - Chat-style layout: outbound messages aligned right (blue bubble), inbound aligned left (gray bubble)
   - Each bubble shows: body text, timestamp, status indicator for outbound (queued/sent/delivered/failed icons)
   - Bottom compose bar: TextField + Send IconButton
   - Send calls smsService.sendP2PMessage, adds message to local list optimistically
   - Messages ordered oldest-first (natural chat scroll), auto-scroll to bottom on load
   - Pagination for older messages (scroll to top loads more)

4. Create `features/sms/compose_message_screen.dart`:
   - Takes optional voterId parameter (pre-selected voter) or shows voter search
   - If no voter pre-selected: search field that calls voter search RPC, show results as selectable list
   - Selected voter shown as chip at top
   - Message body TextField (multiline, max 1600 chars -- 10 SMS segments)
   - Character count indicator
   - Send button: validates voter selected + body not empty, calls smsService.sendP2PMessage
   - On success: navigate to ConversationThreadScreen for that voter
   - On error: show SnackBar, keep message in field for retry

5. Create `features/sms/template_list_screen.dart` (Admin only):
   - Loads templates via smsService.listTemplates()
   - ListView of template cards: name, body preview (truncated), merge fields as chips
   - Tap card -> navigate to TemplateFormScreen(templateId) for editing
   - FAB: create new -> navigate to TemplateFormScreen() (no templateId = create mode)
   - Swipe to delete (soft delete via smsService.deleteTemplate)

6. Create `features/sms/template_form_screen.dart` (Admin only):
   - Takes optional templateId (edit mode if provided, create mode if not)
   - If editing: load template via smsService.getTemplate
   - Fields: name (TextField), body (multiline TextField with hint "Use {{.FirstName}}, {{.LastName}}, etc.")
   - Available merge fields shown as tappable chips below body field: FirstName, LastName, City, District, Party
   - Tapping a chip inserts the merge tag at cursor position in body field
   - Preview button: calls smsService.previewTemplate (or renders locally with sample data)
   - Save button: create or update template
   - Character count for body

7. Create `features/sms/campaign_list_screen.dart` (Admin only):
   - Loads campaigns via smsService.listCampaigns()
   - ListView of campaign cards showing: name, status badge (draft=gray, sending=blue, paused=yellow, completed=green, cancelled=red), progress bar (sentCount/totalRecipients), counts (sent/delivered/failed)
   - Tap card -> navigate to campaign detail (expanded card or bottom sheet showing full stats)
   - FAB: create new -> navigate to CampaignCreateScreen
   - For sending campaigns: show animated progress indicator
   - Pause/cancel actions available via popup menu on sending campaigns

8. Create `features/sms/campaign_create_screen.dart` (Admin only):
   - Step-based form (local enum state, same pattern as door_knock_screen):
     Step 1: Campaign name (TextField)
     Step 2: Select template (dropdown or list from smsService.listTemplates)
     Step 3: Define segment (reuse voter filter UI pattern: district dropdown, party dropdown, tag multi-select -- build segment_filters JSON)
     Step 4: Review (show name, template preview, estimated recipients count)
   - Create button: calls smsService.createCampaign -> shows success with option to Launch Now or Launch Later
   - Launch Now: calls smsService.launchCampaign -> navigate to CampaignListScreen
   - Show 10DLC status check: if not approved, show warning and disable Launch button with explanation
