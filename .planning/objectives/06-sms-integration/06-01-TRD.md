<anti_patterns>
- Never send from raw phone numbers -- always use MessagingServiceSid
- Never process webhooks synchronously -- Twilio expects < 15s response; publish to NATS, return 200
- Never store Twilio auth tokens in plaintext -- use env vars (existing config.Load() pattern)
- Never check quiet hours client-side only -- must be server-side gate
- Never poll for delivery status -- use StatusCallback webhooks
- Never mix P2P and A2P traffic on same Messaging Service SID
</anti_patterns>

<error_recovery>
- Twilio webhook retries: Use twilio_message_sid as idempotency key (UNIQUE constraint + ON CONFLICT DO NOTHING for inserts, UPDATE WHERE for status)
- NATS consumer failure: AckExplicit policy means message redelivers on crash; process must be idempotent
- Suppression race condition: Check suppression at actual send time (in NATS worker), not at queue time
- Quiet hours edge case: All Maine recipients are Eastern Time, simplifying timezone logic for v1
</error_recovery>

<research_context>
- Twilio Go SDK v1.30.4: `github.com/twilio/twilio-go`, typed clients via `twilioApi.CreateMessageParams{}`
- Webhook signature validation: `client.NewRequestValidator(authToken).ValidateBody(url, body, signature)`
- Twilio inbound webhook sends `application/x-www-form-urlencoded` POST with MessageSid, From, To, Body, OptOutType, MessagingServiceSid
- Twilio status callback sends MessageSid, MessageStatus (queued, sent, delivered, undelivered, failed), ErrorCode
- NATS JetStream new API: `jetstream.New(nc)` then `js.CreateOrUpdateStream(ctx, StreamConfig{...})`
- Consumer: pull-based with `cons.Fetch(1)` or `cons.Next(FetchMaxWait(5s))`
- OptOutType values: "STOP", "HELP", "START", or "" (empty = normal message)
- Twilio handles carrier-level STOP automatically; webhook OptOutType param tells us it happened so we can update suppression_list
</research_context>
