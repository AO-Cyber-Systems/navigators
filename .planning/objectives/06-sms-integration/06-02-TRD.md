<objective>
Message templates with merge fields, A2P broadcast campaigns with batch sending, and 10DLC registration gating.

Purpose: Enables Admins to create reusable message templates and launch broadcast campaigns targeting voter segments. Campaign execution happens asynchronously via NATS workers with rate limiting, and A2P sends are gated on 10DLC registration status.

Output: Template CRUD service, campaign lifecycle (draft -> launch -> sending -> completed), NATS batch worker for campaign sends, 10DLC status gating, campaign progress tracking.
</objective>

<anti_patterns>
- Never check suppression at campaign queue time only -- check at actual send time in NATS worker
- Never send A2P without 10DLC approval -- carriers filter with error 30034
- Never mix P2P and A2P on same Messaging Service SID
- Never use unsafe template rendering -- Go text/template with restricted function map
</anti_patterns>

<error_recovery>
- Campaign batch failure: Worker tracks per-message success/failure; campaign continues on individual send failures
- Rate limiting: Use golang.org/x/time/rate at 1 msg/sec conservative default (adjustable after 10DLC approval)
- Template rendering failure: Skip voter, increment failed_count, log error, continue batch
- NATS publish failure on campaign launch: Return error to admin, campaign stays in 'draft' status
</error_recovery>

<research_context>
- Template merge fields use Go `text/template`: `{{.FirstName}}`, `{{.LastName}}`, `{{.City}}`, `{{.District}}`, `{{.Party}}`
- VoterContext struct provides merge field data from voter record
- Campaign batch sends published to NATS subject `navigators.sms.campaign.send`
- Each NATS message = one voter send job (voter_id, campaign_id, template_id)
- Rate limiter: `golang.org/x/time/rate` with rate.NewLimiter(1, 1) -- 1 msg/sec default
- 10DLC status stored in sms_config.ten_dlc_status; gate on 'approved'
- Twilio BrandRegistration + UsAppToPerson APIs for registration (admin tracks status, not automated flow)
- segment_filters JSONB uses same filter format as ListVoters RPC
</research_context>

<gotchas>
- golang.org/x/time/rate is already an indirect dep in go.mod -- may need `go get golang.org/x/time`
- Campaign segment_filters must reuse existing voter filter logic (ListVoters already supports district, party, tags, etc.)
- Template merge_fields TEXT[] column stores field names used in template body for validation/display
- Campaign launch is two-step: CreateCampaign (draft) -> LaunchCampaign (starts sending) to prevent accidental sends
- When campaign completes (sent_count + failed_count >= total_recipients), worker updates status to 'completed'
</gotchas>
