<anti_patterns>
- DO NOT return raw geometry columns from sqlc queries. Always convert to GeoJSON string via ST_AsGeoJSON or individual lat/lng via ST_X/ST_Y. sqlc maps geometry to interface{} otherwise.
- DO NOT use ST_Within for new code -- use ST_Contains (turf contains voter, not voter within turf) for semantic clarity. Both work but ST_Contains(boundary, point) reads better.
- DO NOT skip ST_ForcePolygonCCW when storing boundaries. GeoJSON RFC 7946 requires CCW exterior rings, but user-drawn polygons may have any winding order.
- DO NOT skip NULL/geocode_status filters. Always add `v.location IS NOT NULL AND v.geocode_status = 'success'` in spatial queries.
</anti_patterns>

<error_recovery>
- If sqlc generate fails on PostGIS queries: ensure geometry columns use ST_AsGeoJSON conversion in SELECT, not raw geometry return. The sqlc.yaml overrides geometry->string but only when columns are explicitly typed.
- If ST_GeomFromGeoJSON fails: input GeoJSON may have wrong winding order or invalid coordinates. Wrap with ST_ForcePolygonCCW(ST_GeomFromGeoJSON(...)) and validate SRID=4326.
- If spatial query returns 0 results unexpectedly: check that voters have geocode_status='success' and location IS NOT NULL. Also verify turf boundary actually covers the expected geographic area (not inverted due to winding order).
</error_recovery>

<gotchas>
- sqlc and PostGIS: geometry columns map to interface{} if not converted. Always use ST_AsGeoJSON in SELECT to get string type. Already configured in sqlc.yaml (geometry->string override) but explicit conversion is safer.
- Polygon winding order: Always ST_ForcePolygonCCW on INSERT/UPDATE. Flutter polygon drawing may produce CW coordinates.
- Voter location NULLs: ~20-40% of voters may not be geocoded. All spatial queries MUST filter geocode_status='success'.
- The existing CreateTurf sqlc query does not include boundary. Need a new query (CreateTurfWithBoundary) or modify the existing one.
- Turf assignments already work (AssignUserToTurf, RemoveUserFromTurf). Do not rewrite -- extend TurfInfo with new fields.
</gotchas>
