<anti_patterns>
- DO NOT download all zoom levels. Restrict to 12-18. Lower zooms waste storage; higher zooms have diminishing returns for canvassing.
- DO NOT skip FMTC rate limiting. Set rateLimit to ~100-200 tiles/sec. OSM tile servers ban aggressive downloaders.
- DO NOT set userAgentPackageName to empty string. OSM requires proper identification.
- DO NOT try to use flutter_map_heatmap package. It is INCOMPATIBLE with flutter_map 8. Must use custom Canvas overlay.
- DO NOT load heat map data without bounds. Always pass the current map viewport bounding box to the density grid API.
</anti_patterns>

<error_recovery>
- If FMTC initialise() fails: check that the app has file system access. On iOS, FMTC stores in app documents directory. On Android, app-specific storage. Ensure no stale ObjectBox lock files.
- If tile download stalls or gets HTTP 429: reduce rateLimit to 50 tiles/sec. Consider switching to a commercial tile provider URL.
- If heat map CustomPainter doesn't render: verify that the custom widget is positioned correctly as a FlutterMap child. Use FlutterMapController to get MapCamera for coordinate projection.
- If FMTC CustomPolygonRegion fails: fall back to RectangleRegion using the bounding box of the turf polygon instead.
</error_recovery>

<research_context>
From 03-RESEARCH.md:
- FMTC 10.1.1: FMTCObjectBoxBackend().initialise() before runApp. FMTCStore('turf-{id}'), store.manage.create(), CustomPolygonRegion(outline: turfBoundaryPoints), region.toDownloadable(minZoom: 12, maxZoom: 18, options: TileLayer), store.download.check() for tile count estimate, store.download.startForeground(region, parallelThreads: 5, rateLimit: 200)
- FMTC is GPL-3. Note license in comments. For MVP/pilot this is acceptable; commercial license needed for production release.
- Heat map: server returns grid cells with grid_lat, grid_lng, voter_count, contacted_count, support_count. Flutter renders using CustomPainter with RadialGradient circles. Color based on density (voter_count) or support ratio (support_count/contacted_count).
- Walk list: server returns voters with sequence numbers, ordered by nearest-neighbor. Flutter displays as numbered list with address, distance to next voter.
- Tile storage estimate: ~10-100MB per typical turf at zoom 12-18. Show estimate to user before download.
</research_context>

<gotchas>
- FMTC GPL-3 license: add a comment in tile_cache_service.dart noting the license requirement. Production release needs commercial license.
- FMTC must be initialized BEFORE runApp. If init is async, wrap main() in an async function.
- CustomPolygonRegion needs List of LatLng. Reuse TurfInfo.boundaryPoints from map_service.dart models.
- Heat map MapCamera.project() converts LatLng to screen Offset. This changes on every map move/zoom. Must repaint on camera change.
- OSM tile download at rateLimit: 200 is the max recommended. For a typical turf (~2000 tiles), download takes ~10 seconds.
- Store names must be unique. Use 'turf-{turfId}' pattern to avoid collisions.
</gotchas>
