<anti_patterns>
- DO NOT display prohibited fields in voter profile (no SSN, full DOB, driver license, felony)
- DO NOT allow non-admin users to manage the suppression list
- DO NOT skip suppression check before any outreach operation (this is a legal requirement)
- DO NOT hardcode API URLs in Flutter -- use configuration/environment
- DO NOT block Flutter UI on long operations -- file upload and import should be async with progress indicators
</anti_patterns>

<error_recovery>
- Flutter file upload fails: Show error toast, allow retry. The presigned URL has 5-minute expiry so request a new one on retry.
- Import progress polling: Poll every 2 seconds. If 3 consecutive polls fail (network error), show offline indicator but keep polling.
- Voter search returns error: Show error message in list, allow retry. Don't clear the search field.
- Suppression check fails: Fail closed -- if suppression check errors, do NOT allow the outreach operation. Log the error.
</error_recovery>

<research_context>
From 02-RESEARCH.md:
- Suppression list keyed on voter_id (UUID FK to voters). Simple table with reason, added_by, added_at.
- Suppression check: SELECT EXISTS(SELECT 1 FROM suppression_list WHERE voter_id=$1) -- used before any outreach op.
- Full voter profile fields (permitted per Maine 21-A s196-A): first_name, last_name, middle_name, suffix, year_of_birth, residence address, mailing address, party, voter_status, registration_date, congressional_district, state_senate_district, state_house_district, county, municipality, ward, precinct, voting_history, source_voter_id, geocode_status, tags.
- Eden upload.Service uses presigned PUT URLs to MinIO. Flutter client: (1) call StartImport to get presigned URL, (2) PUT file to presigned URL, (3) call ConfirmUpload to trigger processing, (4) poll GetImportStatus.
</research_context>

<gotchas>
- Flutter needs ConnectRPC client setup. Use the `connectrpc_dart` package if available, or plain HTTP POST with JSON to ConnectRPC endpoints (Connect protocol supports JSON by default).
- File upload in Flutter uses presigned URL (PUT directly to MinIO) -- NOT multipart upload to the Go server. This keeps the Go server out of the data path for large files.
- Voter profile must show voting history from JSONB array. Flutter needs to parse this JSON structure.
- The import screen is admin-only. Use the auth state to check role and conditionally show the import tab.
- Eden Flutter uses Riverpod. Create providers for voter state management.
</gotchas>
