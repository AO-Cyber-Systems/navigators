<anti_patterns>
- DO NOT geocode synchronously during import handler -- always async background
- DO NOT run Census batch requests in parallel -- process sequentially with 5-second delays between batches to avoid overwhelming the free government service
- DO NOT skip TurfScopedFilter on any voter query -- even admin queries should go through it (ScopeAll just means no filter)
- DO NOT return voters without logging audit access -- every search and view must call AuditService.LogVoterAccess
- DO NOT use Google geocoding without a cost ceiling -- track geocoding cost per import and stop if it exceeds a configurable limit
</anti_patterns>

<error_recovery>
- Census Geocoder timeout/500: Retry batch up to 3 times with exponential backoff (5s, 15s, 45s). After 3 failures, mark addresses as geocode_status='failed' and queue for Google fallback.
- Google Maps API error: Log error, mark address as geocode_status='failed'. Do not retry more than once per address per import cycle.
- Search returns no results: Return empty list (not error). Frontend shows "No voters found" message.
- Geography filter with no turfs assigned: TurfScopedFilter returns empty TurfIDs for Navigator with no turfs. Query returns empty results (correct -- they have no assigned area).
</error_recovery>

<research_context>
From 02-RESEARCH.md:
- Census Geocoder batch endpoint: POST multipart to https://geocoding.geo.census.gov/geocoder/locations/addressbatch
  - Form fields: addressFile (CSV: UniqueID,Street,City,State,ZIP), benchmark="Public_AR_Current", returntype="locations"
  - Response CSV: UniqueID, InputAddress, Match, MatchType, MatchedAddress, Coordinates, TigerLineID, Side
  - Coordinates format: "lon,lat" (LONGITUDE FIRST) -- do NOT swap
  - Max 10,000 addresses per batch
- Google Maps Go client: googlemaps.github.io/maps -- has built-in rate limiting (50 QPS), retry
- PostGIS ST_Within for geography filtering: voters.location within turf boundary polygons
- pg_trgm similarity search: `search_text % @query` with `similarity(search_text, @query) AS score` for ranking
- Voter source_voter_id for exact ID lookup (B-tree index)
- Voting frequency = count of entries in voting_history JSONB array
</research_context>

<gotchas>
- Census Geocoder has no official rate limit docs but community reports suggest serial processing with delays. Do NOT parallelize batch requests.
- Census coords are lon,lat (not lat,lon). When creating PostGIS POINT: ST_SetSRID(ST_MakePoint(longitude, latitude), 4326). This matches PostGIS convention (x=lon, y=lat).
- Google Maps client import: googlemaps.github.io/maps (NOT google.golang.org/api/...). Add to go.mod: `go get googlemaps.github.io/maps`
- pg_trgm % operator requires minimum similarity threshold (default 0.3). For short names, this may be too strict. Consider lowering via SET pg_trgm.similarity_threshold = 0.2 or use word_similarity() instead.
- PostGIS ST_Within requires the voter POINT to be non-null. Filter WHERE location IS NOT NULL when doing geographic queries.
- Voting frequency filter needs to count JSONB array elements: jsonb_array_length(voting_history) >= @min_vote_count
</gotchas>

       8. Sleep 5 seconds between Census batches
       9. Repeat until no more pending voters or context cancelled
